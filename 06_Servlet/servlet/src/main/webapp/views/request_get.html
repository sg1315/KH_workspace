Get방식으로 요청한 응답페이지

HTTP요청 메서드중 하나로 서버에서 정보를 가져올 때 사용된다.

특징 1. 요청시 URL의 쿼리 문자열(Query String)을 포함하여 데이터를 전송
=> 사용자가 입력한 값(데이터)들이 url에 노출됨
=> 보안유지가 어려움
=> 즉, 로그인이나 회원가입같은 경우 GET방식은 부적합 함

특징 2. 데이터 길이 제한이 있음
=> 방대한 데이터들을 담았을 경우 초과하는 데이터는 절단되어 넘어감(평균 2048자)
=> 즉, 게시판 작성같은 경우 GET방식 부적합 함

특징3. 캐싱기능및 즐겨찾기 가능
url을 저장해둔 뒤 재요청하는 방식
=> 검색기능에 적합
